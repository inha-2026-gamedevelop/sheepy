# 모델 전환 인수인계 문서

> 목적: 주력 모델이 바뀌는 시점(신모델 출시, 구모델 지원 종료 등)에 진행 중/미완 작업의 현황과 재개 절차를 후속 세션에 넘긴다. | 2026-07-11 이 프로젝트용 판정: **현재 해당 없음(삭제 후보) — 아래 참고.**

## 이 문서가 이 프로젝트에 적용되지 않는 이유

원본 템플릿은 "Fable 5 포함 사용량 종료" 같은 **특정 시점에 발생하는 모델 전환 이벤트**를 전제로, 그 시점의 미완 작업을 다음 모델 체제로 넘기기 위한 문서였다. 이 프로젝트에는 그런 이벤트가 없다 — 세션마다 다른 모델이 쓰일 수 있지만(이번 세션도 `/model sonnet`으로 전환된 이력이 있다), "특정 날짜에 모델 A가 끊기고 모델 B로 전원 이관"하는 상황은 발생하지 않는다.

**이 프로젝트에서 실질적으로 같은 역할을 하는 것은 `claude/PLAN.md`다.** 모델이 무엇이든, 세션이 끊길 때마다 "이번에 고친 것 / 해야 할 것"을 그 문서에 남기는 방식으로 이미 인수인계가 이뤄지고 있다([CHECKPOINT_RULES.md](CHECKPOINT_RULES.md) 참조). 별도의 "모델 전환용" 문서를 유지하는 것은 이 프로젝트 규모에서 이중 관리 비용만 늘린다.

## 권고

이 문서는 **폐기(삭제) 후보**다. `ADAPTATION_GUIDE.md`의 원본 등급표도 "해당 이벤트가 없으면 만들지 않아도 된다"고 명시하고 있다. 다만 실제 삭제는 사용자 확인 후 진행한다(문서 삭제는 되돌리기 쉬운 작업이지만, 파일 존재 자체가 팀에 참고가 될 수도 있어 임의 삭제하지 않음).

## 만약 실제로 모델 전환 이벤트가 생긴다면

아래 골격만 남겨둔다 — 실제 이벤트 발생 시 이 절부터 채운다.

```markdown
## 1. 전환 배경과 후속 모델 체제
- 무엇이 언제 바뀌는가

## 2. 메모리 신선도 경고
git 실측과 메모리 기록을 대조해 낡은 메모리를 표로 경고.

## 3. 진행 중/미완 작업 현황표
claude/PLAN.md의의 미완 항목을 기준으로 작성.

## 4. 새 세션 첫 프롬프트 예시
[CONVERSATION_SPLIT_RULES.md](CONVERSATION_SPLIT_RULES.md)의 이관 형식을 따른다.
```
